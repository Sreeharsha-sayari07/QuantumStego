How The Quantum + Auth Engine Works

STEP 01 — ENCODE

Quantum Key Generation

8 qubits run through H → CNOT → Rz → Measure. Passphrase is hashed (djb2 ⊕ fnv1a) and XOR'd with the quantum key to create a combined seed unique to image + passphrase.
finalSeed = imgSeed ⊕ quantumKey ⊕ passHash
STEP 02 — ENCODE

Quantum LSB Embedding

Fisher-Yates shuffle seeded by finalSeed permutes all pixel indices. Message bits replace pixel LSBs in quantum-randomised order. A unique EOF marker terminates the payload.
pixel[shuffled[i]].R = (R & 0xFE) | bit
STEP 03 — DECODE

Passphrase Verification

The identical circuit re-runs on the stego image seed. If passphrase is wrong → different finalSeed → wrong pixel order → EOF marker never found → authentication fails.
wrong pass → wrong seed → no EOF → FAIL ✗
STEP 04 — DECODE

Authenticated Extraction

Correct passphrase reconstructs exact pixel order. LSBs extracted, bits grouped into bytes, converted to ASCII. EOF found → authentication verified → message revealed.
correct pass → correct seed → EOF ✓
